Projeto Zomba
Desenvolvendo jogos usando tecnologia HTML5 e NodeJS para o Facebook
Este trabalho foi desenvolvido por Danillo Santos, Caio Portella, Driely Alves, Bruno Paiva e o orientador Mauricio Neves Asenjo. Somos alunos da Universidade Santa Cecília e estamos cursando o último ano do curso Ciência da Computação.
O objetivo do trabalho é demonstrar a coleção de recursos tecnológicos que estão disponíveis no HTML5, amplificando a interatividade e conectividade da web se beneficiando do grande desempenho do NodeJS, com o intuito de criar jogos multijogadores para o Facebook.
A razão de utilizarmos essas tecnologias é que juntas elas são capazes de criar aplicações complexas que antes só eram possíveis de serem criadas em desktop. Com o crescimento de desenvolvimento de jogos em grande escala nos últimos anos, decidimos utilizar o “Marketplace” do Facebook.

O protótipo será um jogo inovador de terror e ação em 2D no qual se passa em uma universidade. Um experimento chamado "Zomba" feito por alunos cientistas, tinha tudo para revolucionar o mundo. Porém, por um pequeno erro, ocorreu uma explosão, espalhando um vírus que ao infectar as pessoas causou mutações. Os alunos sobreviventes terão que lutar por suas vidas e encontrar o antivírus antes que seja tarde demais.
O jogo será um aplicativo integrado ao Facebook que possibilitará criar e gerenciar grupos de jogadores, acessar o bate-papo, enviar convites do jogo para os amigos, receber notificações no decorrer do jogo, além de compartilhar seus feitos e pontuações na sua linha do tempo. Tudo isso através da integração com seu perfil nesta rede social. Para isso, usaremos como ferramenta a API disponibilizada pelo Facebook.
O grande motivo para o uso do NodeJS será para fornecer um melhor desempenho em tempo real ao jogo, pelo fato que o mesmo é um interpretador JavaScript do lado do servidor que altera a noção comum de como um servidor Web deveria funcionar. Seu objetivo é possibilitar que um programador crie aplicativos altamente escaláveis e escreva código que manipule dezenas de milhares de conexões simultâneas em uma, e somente uma máquina física.
Para fazermos a comunicação e estabelecer conexões de "soquete" entre um navegador da web e um servidor usaremos a API do HTML5 chamada WebSocket. Em outras palavras, é utilizado para estabelecer uma conexão persistente entre o cliente (jogador) e o servidor, onde ambas as partes podem trocar dados a qualquer momento.
Para o jogo ser visualizado no navegador utilizamos a API do HTML5 chamada Canvas, que torna possível a renderização do jogo em tempo real. E para tornar a experiência de jogo mais envolvente, utilizamos outra API do HTML5 chamada WebAudio, que permite o uso e uma grande capacidade de manipulação de sons no navegador.
